## Why

目前扩展已支持导出，但缺少对应导入能力，用户在重装浏览器、迁移设备或恢复数据时无法把历史剪藏回灌到本地库。先实现“仅导入本应用导出文件”的第一阶段能力，可以快速补齐数据闭环并降低数据丢失风险。

## What Changes

- 在管理界面新增导入入口，支持选择并读取由 Block Clipper 导出的 JSON 数据文件。
- 新增导入校验流程：仅接受本应用导出结构，非法文件给出明确错误提示。
- 新增导入写入流程：将合法记录写入本地存储并刷新列表视图。
- 新增重复处理策略（第一阶段以“保留已有+跳过重复”为默认策略）。
- 新增导入结果反馈（成功数量、跳过数量、失败数量与原因摘要）。

## Capabilities

### New Capabilities
- `data-import`: 支持将 Block Clipper 自身导出的数据文件导入到本地存储，并提供校验、去重与结果反馈。

### Modified Capabilities
- 无

## Impact

- 受影响模块：`entrypoints/sidepanel/*` 或 `public/options.*`（导入入口与结果反馈 UI）、`utils/storage/*`（批量写入与去重辅助）、`utils/exporter/*`（导出结构契约复用）。
- 受影响数据流：新增“文件选择 -> JSON 解析 -> 数据校验 -> 去重写入 -> UI 刷新”的本地导入链路。
- API/依赖影响：不引入后端，不新增远程依赖，优先复用现有 Block 类型与存储服务。

## Success Criteria

- 用户可通过界面选择一个 Block Clipper 导出的 JSON 文件并完成导入。
- 合法数据可被正确写入，导入后在列表中可见。
- 非法文件或结构不匹配时，系统给出明确失败原因且不污染现有数据。
- 重复数据不会被重复写入，并在结果中展示跳过数量。

## Non-goals

- 第一阶段不支持导入第三方格式（如 Notion、Readwise、通用 Markdown 包）。
- 第一阶段不支持压缩包、多文件批量导入、流式超大文件导入优化。
- 不实现云端同步、账号系统、跨设备自动合并。
- 不在本阶段实现复杂冲突策略（如交互式字段级合并）。
