## 🎯 Given-When-Then 格式说明

### 场景结构

每个测试场景包含以下部分：

- **Given (给定)**: 前置条件，测试环境的初始状态
- **When (当)**: 执行的操作或触发的事件
- **Then (那么)**: 预期的结果和验证点

### 示例

```gherkin
场景: 使用快捷键剪藏内容
  Given 用户在浏览器中打开网页
    And 用户选中了文字
  When 用户按下 Ctrl+Shift+Y
  Then 应该显示成功通知
    And 内容被保存到数据库
```

### 好处

1. **易读**: 非技术人员也能理解
2. **结构化**: 清晰的前置条件、操作和预期
3. **可执行**: 直接作为测试检查清单
4. **文档化**: 既是测试也是功能文档

## 📝 测试流程

### 测试前准备

1. **构建扩展**
   ```bash
   pnpm run build
   ```

2. **加载到浏览器**
   - 打开 `chrome://extensions/`
   - 开启"开发者模式"
   - 点击"加载已解压的扩展程序"
   - 选择 `.output/chrome-mv3` 目录

3. **准备测试页面**
   - 打开推荐测试网站
   - 清空之前的测试数据（可选）

### 执行测试

1. **选择测试场景**
   - 从场景文件中选择要测试的场景
   - 或使用打印的检查清单

2. **遵循 Given-When-Then**
   - 确保 Given 条件满足
   - 执行 When 操作
   - 验证 Then 结果

3. **记录结果**
   - 在检查清单上标记 ✅ 或 ❌
   - 记录发现的问题

### 测试后清理

1. **清理测试数据**
   - 打开侧边栏
   - 删除测试剪藏的内容

2. **报告问题**
   - 记录失败的测试
   - 描述重现步骤
   - 附上截图（如果适用）
